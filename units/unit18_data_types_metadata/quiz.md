# クイズ：データの種類と特性

## 問1
次の組み合わせとして最も適切なものはどれですか。

- 顧客からの自由記述メール
- 売上集計テーブル
- AIエージェントが出力するJSON形式の回答データ

A. 構造化 / 非構造化 / 半構造化  
B. 非構造化 / 構造化 / 半構造化  
C. 半構造化 / 構造化 / 非構造化  
D. 非構造化 / 半構造化 / 構造化

正解: B

解説: 自由記述メールは自然言語の非構造化データです。売上集計テーブルは行と列を持つ構造化データです。JSON形式の回答データはキーと値で整理された半構造化データです。

## 問2
非構造化データの処理について、最も適切なものはどれですか。

A. 必ず完全な表形式に変換しないとAIエージェントでは扱えない。  
B. メールや文書の意味理解は、LLMが強みを発揮する領域である。  
C. メタデータを付ける必要はなく、そのまま検索対象にすればよい。  
D. 従来のデータベース検索だけで十分に意味理解できる。

正解: B

解説: LLMは自然言語の意味理解や要約を得意とします。ただし、RAGで使う場合はチャンキングやメタデータ付与が重要です。

## 問3
社員の所属部署やOS環境をAIエージェントが回答に使いたい場合、最も適切な設計はどれですか。

A. 社員情報をすべてナレッジベースに登録し、RAGで検索する。  
B. 社員IDをもとにAPIで社員データベースから取得し、必要な情報だけプロンプトに追加する。  
C. 全社員の情報を毎回プロンプトに含める。  
D. 社員情報は一切扱わない。

正解: B

解説: 社員情報は構造化データであり、APIで必要な情報だけ取得するのが効率的です。RAGは主に非構造化データ参照に向く手法です。

## 問4
商談メモを処理してCRMへ登録する流れとして、最も適切なものはどれですか。

A. 商談メモをAPIで解析し、要約を手動でCRMに入れる。  
B. 商談メモをLLMで要約し、JSON形式で出力し、CRMにはAPIで登録する。  
C. 商談メモをそのままデータベースへ入れ、必要なときだけ読む。  
D. 要約もCRM登録もすべてLLMに一括で任せ、APIは使わない。

正解: B

解説: 商談メモは非構造化データなのでLLMで理解・要約します。出力はJSONなどの半構造化データにし、CRM登録はAPIで定型処理します。

## 短答
メタデータがRAGで重要な理由を一文で説明してください。

模範解答: メタデータはカテゴリや対象条件などの文脈を与え、AIエージェントが必要な文書を検索・参照しやすくするためです。
