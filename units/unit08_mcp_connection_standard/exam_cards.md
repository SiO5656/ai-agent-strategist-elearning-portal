# [[Unit08]] 試験カード

## カード1: MCPの位置づけ
- 問い: MCPはAPIを完全に置き換えるものか。
- 答え: いいえ。APIの上に共通規格をかぶせ、AIエージェントと外部ツールの接続を標準化する考え方。
- キーワード: 接続標準 / 共通フォーマット / APIの完全代替ではない

## カード2: M×N問題
- 問い: MCPが解決するM×N問題とは何か。
- 答え: 複数モデルと複数ツールを個別連携すると、組み合わせ数だけ接続開発・保守が必要になる問題。
- キーワード: モデル数 / ツール数 / 組み合わせ爆発 / 保守コスト

## カード3: ツール選択の自動化
- 問い: MCPによりツール利用はどう変わるか。
- 答え: AIエージェントが利用可能なツールを認識し、依頼内容に応じて必要なツールを選びやすくなる。
- キーワード: 自動選択 / ルーティング / 複数ツール実行

## カード4: ストラテジストの役割
- 問い: 接続方式の判断でストラテジストが整理すべきことは何か。
- 答え: 現在の接続先、将来の追加予定、LLM切り替えの可能性。
- キーワード: ビジネス情報 / アーキテクト連携 / 将来拡張

## カード5: 判断基準
- 問い: 個別APIかMCPかを判断する主な観点は何か。
- 答え: 今の接続数だけでなく、将来ツールやモデルが増える見込みがあるか。
- キーワード: 拡張計画 / 手戻り防止 / 初期コストと保守コスト

## カード6: セキュリティ
- 問い: MCP導入時の注意点は何か。
- 答え: 誰でも作成・公開できるため、提供元の信頼性、権限範囲、接続先データを確認する。
- キーワード: 信頼性 / 権限 / 監査ログ / 情報漏えい
