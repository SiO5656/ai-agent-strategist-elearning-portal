# Unit14 理解度チェック

## 問1
ある企業が、不要な承認工程を残したままAIエージェントで自動化しました。処理速度は上がりましたが、全体の工数はほとんど減っていません。この問題の根本原因として最も適切なものはどれですか。

- A. AIエージェントの性能が不足していた
- B. 現状業務を最適化せず、非効率なプロセスをそのまま自動化した
- C. 自動化対象が狭すぎた
- D. TTS台本が長すぎた

**正解: B**

解説: これは「汚いプロセスの自動化」の典型です。不要な工程を残したまま自動化しても、非効率が高速に回るだけです。

## 問2
ECRSの4観点を、効果の大きい順に並べたものとして正しいものはどれですか。

- A. Simplify → Rearrange → Combine → Eliminate
- B. Combine → Eliminate → Simplify → Rearrange
- C. Eliminate → Combine → Rearrange → Simplify
- D. Rearrange → Combine → Simplify → Eliminate

**正解: C**

解説: まず不要な工程を排除し、次に統合、順序変更、簡素化の順で考えます。

## 問3
「申請フォームの送信と同時に上長へ通知されるようにし、記入と提出の工程を一体化する」は、ECRSのどれに該当しますか。

- A. Eliminate
- B. Combine
- C. Rearrange
- D. Simplify

**正解: B**

解説: 分かれていた複数工程を1つにまとめているため、Combineです。

## 問4
As-Is分析、BPR/ECRS、To-Be設計、AIエージェント適用の順序として適切なものはどれですか。

- A. AI適用 → As-Is分析 → ECRS → To-Be設計
- B. As-Is分析 → BPR/ECRS → To-Be設計 → AI適用
- C. To-Be設計 → AI適用 → As-Is分析 → ECRS
- D. ECRS → AI適用 → As-Is分析 → To-Be設計

**正解: B**

解説: 現状把握、業務見直し、理想設計、AI適用の順序を守ることで、非効率な業務をそのままAI化する失敗を避けられます。
