# Unit06 AIエージェントの起動タイプを判別する

## 対象範囲

- Chapter: Chapter 01
- Source pages: p.35〜p.41
- Unit slug: `agent_trigger_types`

## 学習ゴール

このUnitのゴールは、AIエージェントを「何がきっかけで動き出すか」で分類し、業務シナリオから起動タイプを判定できるようになること。

試験では、用語暗記だけでなく、営業・ヘルプデスク・社内申請などのシナリオを読んで、指示型・定時型・条件型を見分ける問題が出やすい。

## 1. 起動タイプは「きっかけ」の設計

AIエージェントは、同じ処理をしていても、動き出すきっかけが違えば起動タイプが変わる。

- 人がAIエージェントに直接頼む
- 決まった時刻や周期で自動実行する
- システム上の出来事や状態変化を検知して自動実行する

この3つの違いを整理するのが起動タイプ。

## 2. 指示型

指示型は、人間がリアルタイムにAIエージェントへ依頼して動作するタイプ。

例:

- チャットで質問する
- プロンプトを入力する
- ボタンを押して回答生成を依頼する

判定ポイントは「AIエージェントに直接頼んだか」。

## 3. 定時型

定時型は、あらかじめ設定した時刻・曜日・頻度で自動起動するタイプ。

例:

- 毎朝8時にニュース要約を配信する
- 毎週月曜9時に商談進捗レポートを生成する
- 月次で未対応チケットを棚卸する

実行タイミングと処理内容が固定されやすいため、導入前後の作業時間や件数を比較しやすい。

## 4. 条件型

条件型は、特定の出来事や状態変化を検知して自動起動するタイプ。

例:

- 在庫が一定数を下回ったら発注書を生成する
- 問い合わせメールを受信したら分類・回答案作成を始める
- CRMに新規リードが登録されたらウェルカムメールのドラフトを作る

ポイントは、人間の操作が関係していても、それがAIへの直接依頼ではなく業務システム上のイベントなら条件型になり得ること。

## 5. 指示型と条件型の迷いどころ

人が何かを操作したからといって、すべて指示型になるわけではない。

たとえば、営業担当者がCRMに新規リードを登録し、その登録を検知してAIエージェントがウェルカムメールを作るケース。

この場合、営業担当者はAIエージェントに「メールを作って」と頼んでいない。普段通りCRMへ登録しただけ。その登録イベントを条件として、AIエージェントが裏側で起動している。

したがって条件型。

## 6. ヘルプデスクAIでの使い分け

同じヘルプデスクAIでも、場面ごとに起動タイプは変わる。

| 場面 | 起動タイプ | 理由 |
|---|---|---|
| 社員がチャットで質問する | 指示型 | AIエージェントへ直接依頼している |
| 問い合わせメールを受信する | 条件型 | メール受信イベントを検知して起動する |
| 毎朝、未処理チケットを棚卸する | 定時型 | 決まった時間に自動実行する |

1つのAIエージェントが1つの起動タイプに固定されるわけではない。業務の場面ごとに組み合わせて設計する。

## 7. 起動タイプと自律性レベルは別軸

起動タイプは「いつ動くか」。
自律性レベルは「動いた後にどこまで任せるか」。

この2つは独立した設計判断。

例:

- メール受信で起動する → 条件型
- FAQに該当する問い合わせはAIが自動回答し、例外は担当者へ渡す → 協働型

条件型だから必ず自律型になる、という決まりはない。

## 試験対策の覚え方

1. AIへ直接頼んだか → 指示型
2. 時刻・曜日・頻度が決まっているか → 定時型
3. 出来事・状態変化を検知しているか → 条件型
4. 同じAIエージェントでも、場面ごとに複数タイプを組み合わせられる
5. 起動タイプと自律性レベルは別々に判定する
