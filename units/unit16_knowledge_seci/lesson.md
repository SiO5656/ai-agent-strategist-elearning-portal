# [[Unit16]] lesson.md

## タイトル
ナレッジマネジメント：暗黙知・形式知・SECIモデル

## このUnitのゴール
[[AIエージェント]]導入で成果を分けるのは、モデルそのものだけではなく、組織の知識がAIに渡せる形で整っているかどうかです。このUnitでは、[[ナレッジマネジメント]]の基礎、[[暗黙知]]と[[形式知]]の違い、[[RAG]]・[[ファインチューニング]]との関係、そして[[SECIモデル]]の4フェーズを学びます。

## 1. 情報と知識は違う
情報は事実の記述です。購買データ、問い合わせ履歴、マニュアルの記載などは情報として参照できます。知識は、情報をどう解釈し、どのように判断へつなげるかという根拠です。

たとえば「顧客Aの購買データがある」は情報です。一方で「顧客Aは決算前に大型発注する傾向があるため、11月に提案すべきだ」は知識です。[[AIエージェント]]が実務判断を支援するには、単なる情報だけでなく、組織が蓄積してきた判断基準を参照できる状態が必要です。

## 2. ナレッジマネジメントとは
[[ナレッジマネジメント]]とは、個人に属している知識を整理し、共有し、組織として活用できる状態にする取り組みです。

AI導入で重要になる理由は明確です。AIは整理された文書やデータを参照できますが、ベテランの頭の中だけにある勘どころは、そのままでは参照できません。知識が共有されないままだと、AIに使えないだけでなく、異動や退職によって組織の知的資産が失われます。

## 3. 暗黙知と形式知
知識は大きく [[暗黙知]] と [[形式知]] に分けられます。

- [[暗黙知]]: 個人の経験、勘、感覚に基づく知識。言葉や文書として整理されていない。例: 熟練者が機械音から不調を察知する、営業担当者が顧客の反応から提案タイミングを判断する。
- [[形式知]]: 言葉、数値、図表などで表現され、他者が参照できる知識。例: 業務マニュアル、FAQ、判定フロー、対応履歴、社内規程。

[[AIエージェント]]が直接参照できるのは、原則として形式知です。「AIでは対応できない」のではなく、必要な知識が暗黙知のままでAIに渡せない、というケースが多くあります。

## 4. 暗黙知が形式知化されにくい理由
暗黙知が残る理由は、単に時間がないからではありません。重要な障壁は2つです。

1. 言語化の困難さ
   - 熟練者の判断基準は本人にも自覚されにくい。
   - 「マニュアルを書いてください」だけでは引き出せない。
   - 良い問いかけ、インタビュー設計、具体事例の掘り下げが必要です。

2. 知識の囲い込み
   - 「自分しか知らないこと」が存在価値になっている場合、共有が心理的に不利に見える。
   - 知識提供を組織貢献として位置づけ、評価制度や役割に組み込む必要があります。

## 5. ナレッジベースとRAG
暗黙知を形式知化し、既存のマニュアル・FAQ・対応履歴・規程と統合したものが [[ナレッジベース]] です。[[RAG]] は、AIが回答を作る前にナレッジベースを検索し、関連情報を参照して回答を生成する仕組みです。

ここで重要なのは、ナレッジベースの品質が回答品質を直接左右することです。古い情報、誤った情報、粒度の粗い情報が入っていれば、AIはそれをもとに不正確または曖昧な回答を返します。つまり「AIの精度が低い」という問題の多くは、実は知識整備の問題です。

## 6. ファインチューニングとの違い
[[RAG]] は「必要なときに知識を参照させる」方法です。[[ファインチューニング]] は、汎用モデルに自社固有のデータを追加学習させ、用途・文脈・応答スタイルに寄せる方法です。

- 更新頻度が高いFAQ、規程、価格、製品情報: [[RAG]] が向きやすい。
- 自社らしい文体、専門用語、応答トーン: [[ファインチューニング]] が有効な場合がある。

ただし、どちらも前提は同じです。使える形に整理された知識がなければ、参照文書にも学習データにもなりません。

## 7. SECIモデル
[[SECIモデル]] は、暗黙知と形式知が組織内で循環するプロセスを示すフレームワークです。順序は次の4つです。

1. 共同化（Socialization）
   - 暗黙知から暗黙知へ。
   - 一緒に体験することで、言葉になっていない知識が移転する。

2. 表出化（Externalization）
   - 暗黙知から形式知へ。
   - ベテランの経験や勘を、文書、FAQ、判断フロー、AIワークフローへ落とし込む。

3. 連結化（Combination）
   - 形式知から形式知へ。
   - 複数の文書、データベース、ナレッジベースをつなげ、新しい知識体系にする。

4. 内面化（Internalization）
   - 形式知から暗黙知へ。
   - 学習者がマニュアルやAI支援を使いながら実践し、判断を身につける。

## 8. AI時代のSECIモデル
AIエージェントは、SECIモデルの循環を速めます。

- インタビューの文字起こしや構造化で、共同化・表出化を支援する。
- ナレッジベース統合や検索で、連結化を支援する。
- 新人がAIエージェントを使って学ぶことで、内面化を支援する。

この循環が回ると、担当者個人に閉じていた知識が組織の資産になります。AIエージェント導入におけるナレッジマネジメントは、単なる準備作業ではなく、組織の知的競争力そのものです。

## 試験で押さえるポイント
- 情報は事実の記述、知識は判断の根拠。
- AIが参照できる知識は、文書やデータとして整理された形式知。
- 暗黙知の形式知化には、言語化支援と共有を促す組織設計が必要。
- ナレッジベースの品質が [[RAG]] の回答品質を左右する。
- [[SECIモデル]] は `共同化 → 表出化 → 連結化 → 内面化` の順で覚える。
- AIエージェントはSECI循環の媒介役になり、知識の蓄積と活用を加速させる。
