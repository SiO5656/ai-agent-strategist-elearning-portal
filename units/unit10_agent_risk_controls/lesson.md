# [[Unit10]] 講義ノート：AIエージェントのリスクと対策：4カテゴリで整理する

## 学習ゴール
- AIエージェント導入リスクを4カテゴリで判別できる。
- 各リスクに対する基本対策を選べる。
- 「リスクがあるから導入しない」ではなく、段階導入と運用設計で管理する考え方を説明できる。

## 4カテゴリ
1. **信頼性リスク**: 出力や判断が正確でない。ハルシネーション、見落とし、不適切判断。対策はRAG、出典明示、人間確認、自律性レベルの段階引き上げ。
2. **形骸化リスク**: 導入しても現場で使われない。対策は業務フローに自然に組み込む設計、利用状況の定量モニタリング、フィードバック改善。
3. **ブラックボックス化リスク**: 管理されないAIエージェントが増え、目的・データ範囲・責任者が不明になる。対策はAIエージェント台帳、定期棚卸、管理責任の明確化。
4. **セキュリティ・倫理リスク**: 情報漏えい、不正利用、プロンプトインジェクション、バイアス、不公平な判断。対策は入力検査、出力制限、アクセス権管理、データ範囲制御の多層防御。

## 試験での判別軸
- 誤回答・存在しない規程の引用 → 信頼性リスク。
- 現場が使わない・利用率低下 → 形骸化リスク。
- 各部門が独自作成しIT部門が把握できない → ブラックボックス化リスク。
- 機密情報の出力要求・偏った判断 → セキュリティ・倫理リスク。

## 重要な考え方
4カテゴリは連鎖する。誤回答が続くと現場が使わなくなり、放置されたエージェントが管理不在になり、最後に漏えい事故へつながることがある。だから、リスク対策は個別論点ではなく、導入から運用までの全体設計として扱う。
