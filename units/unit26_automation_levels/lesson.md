# [[Unit26]] lesson: 自動化レベルの進化論：Lv.1〜Lv.4の使い分け

## 学習ゴール
- AIエージェント導入を、Lv.1の個人利用からLv.4の自律型まで段階で見分け、MVPで小さく価値検証してからLv.3以降へ進む判断軸を学ぶ。

## 講義ノート
### scene01 Chapter4の入口
- 自動化は一気に全自動化する話ではなく、業務リスクに合う段階を選ぶ設計問題です。まず四つのレベルを地図として持ちます。
- 全自動ありきで考えない
- 業務リスクに合うレベルを選ぶ
- 用語: 自動化レベル = AI活用を自律性と仕組み化の度合いで整理する見取り図。

### scene02 段階導入の理由
- いきなりLv.4を狙うと、成果が出る前に期待値が下がりやすい。小さく試し、信頼とデータを積み上げるほうが現実的です。
- 大きく始めるほど失敗時の反動が大きい
- 小さく始めて信頼を積み上げる
- 用語: MVP = 最小限でも価値を検証できる形で先に試す考え方。

### scene03 Lv.1とLv.2
- Lv.1は必要な時に汎用AIへ都度依頼する段階。Lv.2は用途を決め、プロンプトやナレッジで専門性を持たせたチャットボットです。
- Lv.1は都度利用で再現性が弱い
- Lv.2は専門化するが受動的
- 用語: 仕組み化 = 個人の勘ではなく、同じ使い方を再現できる状態。

### scene04 Lv.3ワークフロー
- Lv.3はトリガーを起点に、事前設計したステップが順番に動く仕組みです。電車のように、決めたレール上では強く安定します。
- トリガーから処理が自動で連鎖
- 強みは再現性と統制しやすさ
- 用語: Lv.3 = 人間が設計したフローを、条件に応じて自動実行する段階。

### scene05 Lv.4自律型
- Lv.4は目標を渡すと、AIエージェントが計画し、実行し、結果を見て次の行動を決めます。柔軟な分、統制と責任分界が難しくなります。
- AIが手順自体を組み立てる
- 柔軟性とリスクが同時に上がる
- 用語: 自律型 = 目標に基づき、計画と実行をAI自身が進める形。

### scene06 境界線の見分け方
- 試験ではツール名より境界線が重要です。仕組み化されているか、自ら起動するか、フロー構造をAIが作ったかで判断します。
- Lv.1と2は仕組み化で判別
- Lv.2と3は自動起動と連携で判別
- Lv.3と4は計画主体で判別
- 用語: 計画主体 = 処理順序を人間が作ったか、AIが作ったかという判定軸。

### scene07 提案書AIの四段階
- 提案書AIでも、都度プロンプト、専用ボット、Salesforce連携ワークフロー、自律提案エージェントの順に段階が上がります。
- 同じ提案書AIでもレベルは変わる
- 連携と自律判断が上がるほどリスクも上がる

### scene08 Lv.2から始めるMVP
- 現実的な初手はLv.2での価値検証です。五名で試し、品質と時間短縮が確認できれば、Lv.3投資の根拠になります。
- まず提案書品質を検証
- 時間短縮データで投資判断
- 用語: 投資判断 = 効果データを根拠に次の自動化レベルへ進むか決めること。

### scene09 Lv.3とLv.4の誤判定
- 自動で動いていても、人間が手順を設計したならLv.3です。AIが何をどの順序で行うかまで計画する場合だけLv.4です。
- 自動実行だけでLv.4とは限らない
- 決め手はフローを誰が作るか

### scene10 覚える軸
- まとめです。自動化レベルは優劣ではなく選択肢です。小さく検証し、信頼とデータを得てから段階的に引き上げます。
- レベルは優劣ではなく適材適所
- MVPで価値を証明してから進む
- 用語: 適材適所 = 業務の性質、リスク、現場負荷に合うレベルを選ぶこと。

## 試験での見分け方
- 具体例を見たら、先に判定軸を言語化してから選択肢を見る。
- ツール名だけで判断せず、起動条件、連携有無、計画主体、処理内容で判別する。
